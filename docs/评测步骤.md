## 🤖 第二步：测试集构建（60 分钟，LLM 辅助）

### 2.1 LLM 批量生成问题（20 分钟）

**Prompt 策略**：为 4 个组设计专用 Prompt，控制问题类型和输出格式

| 组名 | Prompt 核心要求 | 输出字段 |
|------|---------------|---------|
| retrieval_eval | 简单 40%/中等 40%/复杂 20%，答案可直接检索 | query, ground_truth, difficulty, expected_keywords |
| complex_decompose | 必须可拆解为 2-3 个子问题，答案含多步骤 | + expected_sub_questions, answer_structure |
| semantic_chunking | 问题答案分布在文档不同位置，测试切分完整性 | + key_fragment, fragment_location |
| memory_system | 多轮对话形式，标注上下文依赖和记忆类型 | + memory_type, needs_context, refer_to_previous |

**执行逻辑**：
1. 为 25 个 PDF 各生成 200 字摘要（LLM 批量调用，5 分钟）
2. 按文档所属组，选择对应 Prompt，调用 LLM 生成问题（10 个/次批量调用，10 分钟）
3. 解析 JSON 输出，合并为 `test_set_raw.json`（5 分钟）

### 2.2 自动校验（15 分钟）

**规则校验**（快速过滤）：
- 答案长度：10-500 字，否则标记
- 关键词匹配：expected_keywords 必须出现在 ground_truth 或文档中
- 问题格式：必须含疑问词或问号
- 重复检测：Embedding 相似度>0.95 则标记重复

**LLM 自评**（深度过滤）：
- Prompt：评估问题准确性/完整性/可检索性（1-5 分）
- 过滤标准：三项均≥3 分才保留
- 批量调用：10 个问题/次，节省时间

**输出**：`test_set_validated.json`（预期保留 125+ 个问题）

### 2.3 抽样人工校验（20 分钟）

**抽样策略**（分层抽样 30 个问题）：
- 按组：retrieval(12) + complex(8) + semantic(6) + memory(4 场景)
- 按难度：简单 30% + 中等 50% + 复杂 20%
- 按文档：确保每个文档至少 1 个问题被抽检

**校验标准**：
- 5 分：问题清晰 + 答案准确完整 → 直接采纳
- 4 分：小瑕疵但不影响评估 → 采纳
- 3 分：中等问题 → LLM 辅助修正后采纳
- ≤2 分：严重问题 → 剔除

**输出**：`test_set_final.json`（预期 118 个有效问题）

## 📊 第四步：自动化评估（30 分钟）

### 4.1 按组计算指标

| 组名 | 核心指标 | 计算方法 |
|------|---------|---------|
| retrieval_eval | Recall@10, MRR, Faithfulness | expected_keywords 匹配 + RAGAS |
| complex_decompose | 回答完整度，子问题拆分准确率 | LLM-as-Judge 评估覆盖度 |
| semantic_chunking | 信息保留率，检索准确率 | key_fragment 匹配 + 模糊匹配 |
| memory_system | 上下文准确率，响应速度提升，记忆命中率 | 规则检查 + LLM 评估指代理解 |

### 4.2 整体指标汇总
```
整体 Faithfulness = 加权平均(各组 faithfulness × 问题数) / 总问题数
整体 Recall@10 = 仅 retrieval_eval 组计算（其他组不适用）
P95 延迟 = 所有 150 个 query 的延迟 95% 分位线
平均 token = 所有 query 的 token 消耗平均值
```

## 💡 面试话术要点

### 架构设计
```
1. 为什么拆分 RAG 层和 Agent 层？
   → 职责分离：RAG 层专注检索质量，Agent 层专注推理流程
   → 可独立迭代：RAG 优化不影响 Agent 逻辑，反之亦然
   → 可复用：RAG Server 可被多个 Agent/客户端调用
```

### 评估体系
```
2. 为什么分层测评？
   → 不同任务需要不同评估标准，分层能精准定位瓶颈
   → RAG 层：Recall/MRR 测检索，信息保留率测切分
   → Agent 层：Faithfulness 测生成，完整度测推理
   → 避免"平均数掩盖问题"

3. 如何保证测试集质量？
   → LLM 生成 + 规则校验 + LLM 自评 + 抽样人工，三重过滤
   → 118 个问题人工校验通过率 94%
```

### 消融实验
```
4. 消融实验怎么做的？
   → RAG 层：baseline(recursive+dense) → partial(semantic+hybrid) → full(+rerank)
   → Agent 层：baseline → +分解 → +改写 → +记忆
   → 每层独立对比，量化各模块贡献

5. 各模块贡献是多少？
   → RAG 层：语义切分贡献信息保留率 +29%，混合检索贡献 Recall +5%
   → Agent 层：查询分解贡献完整度 +41%，记忆系统贡献延迟 -34%
```

### 成本控制
```
6. 如何控制成本？
   → 分层增强(20% 关键内容)+缓存复用(40% 命中)+预算守护
   → 25 个文档总成本<15 元
   → 评估流程自动化率>95%
```

### 工程价值
```
7. 工程价值？
   → MCP 协议标准化，任何 AI 工具都可调用
   → 评估框架可复用，新业务接入快速验证
   → 双层架构可独立扩展
```
